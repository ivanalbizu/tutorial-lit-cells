@customElement('my-greeting')
Registra el tag en el navegador. A partir de aquí, el navegador sabe qué hacer al encontrar <my-greeting>.

extends LitElement hereda reactividad y Shadow DOM automático

src/my-greeting.ts

Añadir una propiedad reactiva


@customElement('my-greeting')
export class MyGreeting extends LitElement {

  @property({ type: String, reflect: true })
  name = 'Mundo';

}

type: String — convierte el atributo HTML al tipo correcto

reflect: true — sincroniza la propiedad JS con el atributo HTML.
Si cambias la propiedad por código, el atributo en el DOM se actualiza también.

src/my-greeting.ts

Estilos encapsulados en el Shadow DOM


@customElement('my-greeting')
export class MyGreeting extends LitElement {

  @property({ type: String, reflect: true })
  name = 'Mundo';

  static styles = css`
    :host { display: block; padding: 1rem; }
    .highlight { color: #646cff; font-weight: bold; }
  `;

}

:host apunta al propio elemento <my-greeting> en el DOM

Los estilos están dentro del Shadow DOM: no afectan al resto de la página y los estilos externos tampoco los sobreescriben.

src/my-greeting.ts

El método render()


@customElement('my-greeting')
export class MyGreeting extends LitElement {

  @property({ type: String, reflect: true })
  name = 'Mundo';

  static styles = css`
    :host { display: block; padding: 1rem; }
    .highlight { color: #646cff; font-weight: bold; }
  `;

  render() {
    return html`
      <p>Hola, <span class="highlight">${this.name}</span>!</p>
    `;
  }
}

Cada vez que name cambie, Lit llama a render() y actualiza solo la parte del DOM que cambió.

Componente completo

Estructura que repetiremos en todos los componentes


import { LitElement, html, css } from 'lit';
import { customElement, property } from 'lit/decorators.js';

@customElement('my-greeting')
export class MyGreeting extends LitElement {

  @property({ type: String, reflect: true })
  name = 'Mundo';

  static styles = css`
    :host { display: block; padding: 1rem; }
    .highlight { color: #646cff; font-weight: bold; }
  `;

  render() {
    return html`
      <p>Hola, <span class="highlight">${this.name}</span>!</p>
    `;
  }
}

Importar el componente

src/main.ts — registrar el tag


import './my-greeting.js';

El import ejecuta el decorador @customElement y registra el tag en el navegador

Shadow DOM en DevTools

El HTML interno está encapsulado

<my-greeting>

    <style>:host { display: block; ... }</style>

Shadow DOM nativo del navegador — Lit lo usa y lo gestiona por nosotros

Los estilos también viven dentro de la shadow root. Intentar sobreescribir .highlight desde CSS global no tiene efecto.

Resumen

  • @customElement('my-greeting') — registra el tag en el navegador
  • extends LitElement — hereda reactividad y Shadow DOM
  • @property() — propiedad reactiva pública
  • reflect: true — sincroniza propiedad JS ↔ atributo HTML
  • static styles = css`...` — estilos encapsulados
  • render() — Lit lo llama al cambiar las propiedades

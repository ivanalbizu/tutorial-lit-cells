Paso 3 — Propiedades y estado reactivo

@property vs @state · tipos · reflect

Pública vs privada

La diferencia conceptual entre @property y @state

@property

El mundo exterior puede leerla y modificarla:

Atributos HTML, JavaScript externo, componentes padre

@state

Solo el propio componente la conoce y modifica

Estado interno, privado

Ambas disparan un re-render automático al cambiar. Esa es la reactividad de Lit.

my-counter.ts

Imports — añadimos state


import { LitElement, html, css } from 'lit';
import { customElement, property, state } from 'lit/decorators.js';

property y state son dos decoradores distintos con comportamientos distintos

my-counter.ts

Propiedades públicas — tipos primitivos


@customElement('my-counter')
export class MyCounter extends LitElement {

  @property({ type: String })
  label = 'Contador';

  @property({ type: Number })
  step = 1;

}

type: Number — cuando el HTML pone step="2", el atributo llega como string "2".
Lit lo convierte a número 2 automáticamente.

my-counter.ts

Estado interno con @state


@customElement('my-counter')
export class MyCounter extends LitElement {

  @property({ type: String })
  label = 'Contador';

  @property({ type: Number })
  step = 1;

  @state()
  private _count = 0;

}

El prefijo _ señala que es privado por convención

El valor del contador es estado interno: nadie fuera del componente debería poder cambiarlo directamente.

my-counter.ts

Métodos y render()


  // ... propiedades anteriores ...

  private _increment() { this._count += this.step; }
  private _decrement() { this._count -= this.step; }

  render() {
    return html`
      <p>${this.label}</p>
      <div class="counter">
        <button @click=${this._decrement}>-</button>
        <span>${this._count}</span>
        <button @click=${this._increment}>+</button>
      </div>
    `;
  }

@click=${this._decrement} — cuando cambia _count (@state), Lit re-renderiza solo el <span>

my-user-card.ts

Tipos: String, Number, Boolean + reflect


@customElement('my-user-card')
export class MyUserCard extends LitElement {

  @property({ type: String })
  name = 'Anonimo';

  @property({ type: Number })
  age = 0;

  @property({ type: Boolean, reflect: true })
  active = false;

  @state()
  private _showDetails = false;

}

reflect: true en el booleano: cuando active es true, el atributo active aparece en el elemento HTML.
Permite seleccionarlo con CSS desde fuera: [active] { ... }

my-user-card.ts

Aprovechando reflect en los estilos


  static styles = css`
    :host { display: block; border: 1px solid #ccc;
            padding: 1rem; border-radius: 8px; max-width: 300px; }
    :host([active]) { border-color: #646cff; }
    button { margin-top: 0.5rem; cursor: pointer; }
  `;

:host([active]) — aprovecha que active tiene reflect: true:
cuando la propiedad es true, el selector captura el elemento y cambia el borde a morado.

my-user-card.ts

Render — combinando todos los tipos


  render() {
    return html`
      <h3>${this.name}</h3>
      <span>${this.active ? 'Activo' : 'Inactivo'}</span>
      <button @click=${() => this._showDetails = !this._showDetails}>
        ${this._showDetails ? 'Ocultar' : 'Ver detalles'}
      </button>
      ${this._showDetails
        ? html`<p>Edad: ${this.age}</p>`
        : ''}
    `;
  }

@state _showDetails controla la visibilidad interna sin exponer nada al exterior

Boolean attribute: active sin valor = true.
Sin el atributo = false.

Reactividad en tiempo real

DevTools > Console


// Cambiar step del contador en tiempo real

// Activar la segunda tarjeta por código
document.querySelector('my-user-card').active = true;

Sin recargar la página — Lit detecta el cambio en la propiedad y re-renderiza automáticamente

Resumen

  • @property() — propiedad pública, modificable desde fuera
  • @state() — estado privado, solo interno
  • Ambas disparan re-render automático al cambiar
  • type: Number / Boolean — convierte atributos HTML al tipo JS correcto
  • reflect: true — sincroniza propiedad → atributo, útil para selectores CSS
  • Boolean en HTML: presente = true, ausente = false

Paso 4 — Templates y los 7 tipos de binding

Enlazar datos con el DOM de forma declarativa

¿Qué es un binding?

Una conexión entre JavaScript y el DOM

Expresión JS

→

DOM

Cuando la expresión cambia, Lit actualiza solo las partes afectadas del DOM

Hay 7 tipos de binding, dependiendo de qué parte del DOM quieres afectar

1 — Interpolación de texto

${expresion} dentro del template


// Propiedad del componente
html`<p>${this.title}</p>`

// Expresión aritmética
html`<p>${2 + 2}</p>`

// Cálculo con datos
html`<p>${this.items.length} tareas</p>`

La expresión puede ser una propiedad, una operación o cualquier valor JavaScript. Si devuelve otro template html, lo anida.

2 — Binding de atributo

attr=${valor} — sin comillas alrededor


html`<input type="text" placeholder=${this.hint}>`

html`<div class=${this.cssClass}>`

html`<img src=${this.imageUrl} alt=${this.description}>`

Asigna el valor de la expresión a un atributo HTML.
Lit serializa el valor a string si es necesario.

Funciona para cualquier atributo: class, src, href, placeholder...

3 — Binding de propiedad

.prop=${valor} — prefijo punto


html`<input .value=${this.inputValue}>`

html`<my-counter .step=${this.increment}>`

html`<canvas .width=${600} .height=${400}>`

Asigna directamente a la propiedad JavaScript del elemento, no al atributo HTML.
Un atributo siempre es string — una propiedad puede ser cualquier tipo.

Úsalo para pasar objetos, arrays, números, o comunicarte con un Web Component hijo

4 — Binding de evento

@evento=${handler} — prefijo arroba


html`<button @click=${this._handleClick}>Enviar</button>`

html`<input @input=${(e) => this._value = e.target.value}>`

html`<form @submit=${this._onSubmit}>`

El nombre del evento sin el prefijo on: @click, @input, @submit...

Lit gestiona el addEventListener y removeEventListener automáticamente.
Al eliminar el componente del DOM, los listeners también se limpian.

5 — Binding booleano

?attr=${bool} — prefijo interrogación


html`<button ?disabled=${!this.isValid}>Guardar</button>`

html`<details ?open=${this.expanded}>`

html`<input ?required=${this.mandatory}>`

Si la expresión es verdadera: el atributo aparece sin valor (disabled, required...).
Si es falsa: el atributo desaparece por completo.

Es la forma correcta de manejar atributos que en HTML funcionan por presencia o ausencia

6 — Condicional con nothing

Ternario JS + el valor especial nothing


import { nothing } from 'lit';

// Ternario simple
html`${this.isLoggedIn
  ? html`<p>Bienvenido</p>`
  : html`<p>Inicia sesión</p>`
}`

// Con nothing — no renderiza nada, ni comentario en el DOM
html`${this.showBadge ? html`<span class="badge">Nuevo</span>` : nothing}`

nothing es más limpio que '' o null: no deja ningún nodo residual en el DOM.

7 — Listas con .map()

JavaScript puro dentro del template


html`
  <ul>
    ${this.items.map(item => html`
      <li>${item.name}</li>
    `)}
  </ul>
`

No hay directiva especial: es .map() de JavaScript devolviendo un template por cada elemento

Para listas grandes donde el rendimiento importa, existe la directiva repeat.
Para la mayoría de casos, map es suficiente y más sencillo.

my-task-list.ts

Estructura base — interfaz y estado


import { LitElement, html, css, nothing } from 'lit';
import { customElement, state } from 'lit/decorators.js';

interface Task {
  id: number;
  text: string;
  done: boolean;
}

@customElement('my-task-list')
export class MyTaskList extends LitElement {
  @state()
  private _tasks: Task[] = [
    { id: 1, text: 'Aprender Lit', done: true },
    { id: 2, text: 'Crear componentes', done: false },
    { id: 3, text: 'Publicar la app', done: false },
  ];

  @state()
  private _showDone = true;

my-task-list.ts

Getter filtrado y método toggle


  private get _visibleTasks() {
    return this._showDone
      ? this._tasks
      : this._tasks.filter(t => !t.done);
  }

  private _toggle(id: number) {
    this._tasks = this._tasks.map(t =>
      t.id === id ? { ...t, done: !t.done } : t
    );
  }

No mutar el array directamente.
this._tasks = this._tasks.map(...) — reasignar la propiedad para que Lit detecte el cambio comparando la referencia.

my-task-list.ts

Render — los 7 bindings en acción


  render() {
    const pending = this._tasks.filter(t => !t.done).length;
    return html`
      <h2>Tareas <span>(${pending} pendientes)</span></h2>
      <label>
        <input type="checkbox" .checked=${this._showDone}
          @change=${() => this._showDone = !this._showDone}>
        Mostrar completadas
      </label>
      <ul>
        ${this._visibleTasks.map(task => html`
          <li>
            <input type="checkbox" .checked=${task.done}
              @change=${() => this._toggle(task.id)}>
            <span class=${task.done ? 'done' : ''}>${task.text}</span>
            ${task.done ? html`<small>✓</small>` : nothing}
          </li>
        `)}
      </ul>
      ${this._visibleTasks.length === 0
        ? html`<p><em>No hay tareas pendientes.</em></p>`
        : nothing}
    `;
  }

Resumen — los 7 bindings

  • Texto: ${expr} — inserta valor como texto
  • Atributo: attr=${expr} — asigna al atributo HTML como string
  • Propiedad: .prop=${expr} — asigna a la propiedad JS (cualquier tipo)
  • Evento: @evento=${handler} — enlaza un listener
  • Booleano: ?attr=${bool} — presencia o ausencia del atributo
  • Condicional: ternario JS + nothing para no renderizar nada
  • Lista: .map() para convertir arrays en templates

Nunca mutar un array u objeto directamente — reasignar la propiedad para que Lit detecte el cambio

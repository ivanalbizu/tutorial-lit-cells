Paso 6 — Ciclo de vida de un componente

constructor → connectedCallback → render → disconnectedCallback

Orden del ciclo de vida

De la creación a la eliminación

  constructor()           Al crear la instancia
      ↓
  connectedCallback()     Al insertar en el DOM
      ↓
  willUpdate(changed)     Antes de cada render
      ↓
  render()                Produce el template
      ↓
  firstUpdated()          Tras el primer render (una vez)
      ↓
  updated(changed)        Tras cada render
      ↓
  disconnectedCallback()  Al remover del DOM

Los métodos con super obligatorio: connectedCallback y disconnectedCallback

constructor()

El primero en ejecutarse


import { LitElement, html, css } from 'lit';
import { customElement, property } from 'lit/decorators.js';

@customElement('my-clock')
export class MyClock extends LitElement {
  @property({ type: String })
  label = 'Hora actual';

  private _timerId: number | null = null;
  private _currentTime = '';

  constructor() {
    super();
    this._currentTime = this._formatTime(new Date());
  }
}

super() debe ser la primera línea.
El Shadow DOM todavía no existe aquí.

constructor()

¿Qué hacer y qué no hacer?

Inicializar propiedades internas con valores por defecto

NO acceder a this.shadowRoot ni a elementos del template.
El Shadow DOM se crea después del constructor.

connectedCallback()

El elemento se inserta en el DOM


connectedCallback() {
  super.connectedCallback();
  this._timerId = window.setInterval(() => {
    this._currentTime = this._formatTime(new Date());
    this.requestUpdate();
  }, 1000);
}

super.connectedCallback() es obligatorio

connectedCallback()

¿Por qué requestUpdate()?

_currentTime es una propiedad privada.
Lit no la observa automáticamente.
this.requestUpdate() fuerza un nuevo render.

Aquí también puedes: suscribirte a eventos globales, cargar datos, iniciar animaciones.

willUpdate(changedProps)

Antes de cada render — derivar estado


willUpdate(changedProps: Map<string, unknown>) {
  if (changedProps.has('label') && this.label === '') {
    this.label = 'Sin etiqueta';
  }
}

changedProps contiene los valores anteriores, no los nuevos.
Para leer el nuevo valor usa this.label directamente.

Los cambios aquí no disparan un render adicional

render()

Puro: solo lee estado, retorna template


render() {
  return html`
    <div class="clock">
      <span class="label">${this.label}</span>
      <span class="time">${this._currentTime}</span>
    </div>
  `;
}

Nunca modifiques propiedades dentro de render().
Si tiene efectos secundarios, tu componente se vuelve impredecible.

firstUpdated()

Una sola vez — el Shadow DOM ya existe


firstUpdated() {
  // El Shadow DOM ya existe aquí
  const timeEl = this.shadowRoot!.querySelector('.time');
  console.log('Elemento .time encontrado:', timeEl);
}

Equivale al mounted de otros frameworks.
Ideal para integrar librerías de terceros, enfocar campos de formulario o medir el DOM.

updated(changedProps)

Tras cada render — DOM ya actualizado


updated(changedProps: Map<string, unknown>) {
  if (changedProps.has('label')) {
    const prev = changedProps.get('label');
    console.log(`Label cambió de "${prev}" a "${this.label}"`);
  }
}

Si modificas una propiedad observada dentro de updated(),
causarás otro render. Siempre protege con una condición.

disconnectedCallback()

El elemento se elimina del DOM — limpiar todo


disconnectedCallback() {
  super.disconnectedCallback();
  if (this._timerId !== null) {
    window.clearInterval(this._timerId);
    this._timerId = null;
  }
}

Sin esta limpieza, el setInterval seguirá corriendo en memoria
aunque el elemento ya no esté en el DOM. Memory leak garantizado.

También cancela peticiones de red, remueve event listeners globales

changedProps: el Map explicado

Valores anteriores, no los nuevos


// Supón que label era "Hora" y pasa a ser "Tiempo"
willUpdate(changedProps: Map<string, unknown>) {
  // changedProps.get('label') === "Hora"  // valor anterior
  // this.label                === "Tiempo" // valor nuevo

  if (changedProps.has('label')) {
    const prevLabel = changedProps.get('label') as string;
    console.log(`Cambió de "${prevLabel}" a "${this.label}"`);
  }
}

changedProps existe tanto en willUpdate como en updated.
Ambos reciben el mismo Map con los valores anteriores.

Componente demo: my-lifecycle-demo

Montar y desmontar para ver el ciclo completo


import { LitElement, html, css } from 'lit';
import { customElement, state } from 'lit/decorators.js';
import './my-clock.js';

@customElement('my-lifecycle-demo')
export class MyLifecycleDemo extends LitElement {
  @state()
  private _mounted = true;

  render() {
    return html`
      <div class="controls">
        <button @click=${this._toggle}>
          ${this._mounted ? 'Desmontar reloj' : 'Montar reloj'}
        </button>
      </div>
      <div class="stage">
        ${this._mounted
          ? html`<my-clock label="Demo lifecycle"></my-clock>`
          : html``}
      </div>
    `;
  }

  private _toggle() {
    this._mounted = !this._mounted;
  }
}

Orden en la consola

Al montar el reloj

  [constructor]        Se ejecuta antes de conectar al DOM
  [connectedCallback]  El elemento se acaba de insertar
  [willUpdate]         changedProps: Map(0)
  [render]             Se ejecuta para producir el template
  [firstUpdated]       Primer render completado
  [updated]            DOM actualizado

Al desmontar: solo aparece [disconnectedCallback].
Al volver a montar: ciclo completo de nuevo (instancia nueva).

Resumen

  Método               Cuándo              super obligatorio
  ─────────────────────────────────────────────────────────
  constructor()        Al crear instancia  Sí
  connectedCallback()  Al insertar en DOM  Sí
  willUpdate()         Antes de render     No
  render()             Cada actualización  No aplica
  firstUpdated()       Tras 1er render     No
  updated()            Tras cada render    No
  disconnectedCallback()  Al remover       Sí

Regla práctica: inicia en connectedCallback, limpia en disconnectedCallback, accede al DOM en firstUpdated, deriva estado en willUpdate.

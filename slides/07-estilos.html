static styles

Estilos encapsulados en Shadow DOM


import { LitElement, html, css } from 'lit';
import { customElement } from 'lit/decorators.js';

@customElement('my-card')
export class MyCard extends LitElement {
  render() {
    return html`
      <div class="card">
        <slot></slot>
      </div>
    `;
  }

  static styles = css`
    .card {
      border: 1px solid #444;
      border-radius: 8px;
      padding: 1.5rem;
      background: #1e1e1e;
      color: #fff;
    }
  `;
}

static styles

¿Por qué encapsulados?

Los estilos viven dentro del Shadow DOM.
La clase .card solo existe dentro de este componente.

Si en tu página tienes otra clase .card global,
no habrá ningún conflicto.

Usa el tagged template css importado desde 'lit'

Array de estilos

Compartir CSS base entre componentes


// src/shared-styles.ts
import { css } from 'lit';

export const baseStyles = css`
  * {
    box-sizing: border-box;
  }
  :host {
    display: block;
  }
`;

Array de estilos

Importar y combinar en my-card


import { baseStyles } from './shared-styles.js';

// En la clase:
static styles = [
  baseStyles,
  css`
    .card {
      border: 1px solid #444;
      border-radius: 8px;
      padding: 1.5rem;
      background: #1e1e1e;
      color: #fff;
    }
  `
];

Cada componente sigue aislado, pero reutiliza las reglas comunes sin duplicar código

:host

El propio elemento como selector


// En la clase, propiedad con reflect:
@property({ type: String, reflect: true })
variant: 'default' | 'primary' | 'danger' = 'default';
          

:host {
  display: block;
}

:host([variant="primary"]) .card {
  border-color: #4a9eff;
  background: #0d2a4a;
}

:host([variant="danger"]) .card {
  border-color: #ff4a4a;
  background: #4a0d0d;
}

:host(:hover) .card {
  border-color: #888;
}

:host y reflect

Cómo funciona la sincronización

reflect: true sincroniza el valor de la propiedad
como atributo en el HTML del elemento.

Sin reflect, el atributo variant no existe en el DOM
y :host([variant="primary"]) nunca coincide.

::slotted()

Estilizar contenido proyectado


render() {
  return html`
    <div class="card">
      <div class="card-header">
        <slot name="header"></slot>
      </div>
      <div class="card-body">
        <slot></slot>
      </div>
    </div>
  `;
}
          

::slotted(h2) {
  margin: 0;
  font-size: 1.2rem;
  color: #4a9eff;
}

::slotted(p) {
  color: #ccc;
  line-height: 1.6;
}

::slotted() — limitación

Solo hijos directos del slot

::slotted(selector) solo funciona con hijos directos.
No llega a descendientes más profundos.

Si proyectas un <div> que contiene un <p>,
::slotted(p) no alcanzará ese párrafo interno.

Para estilos más profundos, usa CSS custom properties

::part()

Exponer partes internas para el exterior


return html`
  <div class="card" part="card">
    <div class="card-header" part="header">
      <slot name="header"></slot>
    </div>
    <div class="card-body" part="body">
      <slot></slot>
    </div>
  </div>
`;

::part() — uso desde fuera

El usuario del componente estiliza las partes


my-card::part(header) {
  background: #2a1a4a;
  padding: 0.5rem 1.5rem;
  border-radius: 8px 8px 0 0;
}

my-card::part(body) {
  padding: 1.5rem;
}

El autor del componente decide qué partes son públicas.
El usuario las personaliza sin romper la estructura interna.

CSS custom properties

API de theming que atraviesa el Shadow DOM


/* Dentro del componente my-card */
.card {
  border: 1px solid var(--card-border-color, #444);
  border-radius: var(--card-radius, 8px);
  padding: var(--card-padding, 1.5rem);
  background: var(--card-bg, #1e1e1e);
  color: var(--card-color, #fff);
}

var(--nombre, valor-default)
Si la variable no está definida, se usa el valor por defecto.

CSS custom properties

El usuario aplica el tema desde fuera


@customElement('my-theme-demo')
export class MyThemeDemo extends LitElement {
  render() {
    return html`
      <div class="demo">
        <my-card>
          <h2 slot="header">Tarjeta temática</h2>
          <p>Esta tarjeta usa custom properties para su tema.</p>
        </my-card>
      </div>
    `;
  }

  static styles = css`
    .demo {
      --card-bg: #1a1a2e;
      --card-border-color: #e94560;
      --card-radius: 16px;
      --card-color: #eee;
    }
  `;
}

Las variables CSS atraviesan el Shadow DOM de forma nativa

Resumen

  static styles = css`...`     Encapsulados en Shadow DOM
  [baseStyles, css`...`]       Reutilizar CSS base
  :host                        El propio elemento
  :host([attr])                Variantes (requiere reflect: true)
  ::slotted(selector)          Hijos directos proyectados
  part="nombre"                Exponer partes internas
  my-card::part(nombre)        Estilizar desde fuera
  var(--prop, default)         API de theming nativa

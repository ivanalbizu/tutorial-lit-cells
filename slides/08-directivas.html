Paso 8 — Directivas de Lit

repeat · classMap · styleMap · when · guard · ifDefined

¿Qué son las directivas?

Funciones especiales para los templates

Las directivas son funciones que controlan cómo Lit actualiza el DOM.
Se usan dentro de los templates html`...`.

Cada directiva resuelve un problema concreto:
Listas eficientes, clases condicionales, renderizado lazy, memorización, atributos opcionales.

Se importan individualmente — solo pagas por lo que usas

Componente demo

Estructura base para las directivas


import { LitElement, html, css } from 'lit';
import { customElement, state } from 'lit/decorators.js';

interface Person {
  id: number;
  name: string;
  age: number;
  active: boolean;
  profileUrl?: string;
}

@customElement('my-directive-demo')
export class MyDirectiveDemo extends LitElement {
  @state()
  private _people: Person[] = [
    { id: 1, name: 'Ana',   age: 28, active: true  },
    { id: 2, name: 'Luis',  age: 34, active: false },
    { id: 3, name: 'María', age: 22, active: true  },
  ];

  @state() private _highlight = false;
  @state() private _filterActive = false;
}

repeat()

Listas con claves — reordenamiento eficiente


import { repeat } from 'lit/directives/repeat.js';

render() {
  const people = this._filterActive
    ? this._people.filter(p => p.active)
    : this._people;

  return html`
    <ul>
      ${repeat(
        people,
        (person) => person.id,
        (person) => html`<li>${person.name} — ${person.age} años</li>`
      )}
    </ul>
  `;
}

repeat() — clave de rendimiento

Tres argumentos: array · keyFn · templateFn

La función clave retorna un identificador único por elemento.
Cuando el array se reordena, Lit mueve los nodos existentes
en lugar de destruirlos y recrearlos.

¿Cuándo usar repeat vs .map()?

Si la lista solo crece al final, .map() es suficiente.
Usa repeat cuando hay inserciones en el medio, reordenamiento o eliminaciones.

classMap()

Clases CSS condicionales


import { classMap } from 'lit/directives/class-map.js';

// Dentro del template de cada persona:
html`
  <li class=${classMap({
    'person-item':    true,
    'is-active':      person.active,
    'is-highlighted': this._highlight,
  })}>
    ${person.name} — ${person.age} años
  </li>
`

Las claves son nombres de clase CSS, los valores son booleanos.
true → la clase se añade. false → se elimina.

Mucho más legible que construir una cadena de texto manualmente

styleMap()

Estilos inline dinámicos


import { styleMap } from 'lit/directives/style-map.js';

// Dentro del template de cada persona:
html`
  <li style=${styleMap({
    opacity:    person.active ? '1' : '0.4',
    fontWeight: person.active ? 'bold' : 'normal',
  })}>
    ${person.name} — ${person.age} años
  </li>
`

Las claves usan camelCase (estándar de JavaScript).
font-weight en CSS → fontWeight en el objeto.

Úsalo cuando los valores son dinámicos y no pueden resolverse con clases predefinidas

when()

Condicional lazy — evalúa solo lo necesario


import { when } from 'lit/directives/when.js';

// En el render principal:
${when(
  this._people.length === 0,
  () => html`<p class="empty-state">No hay personas en la lista.</p>`,
  () => html`<p class="count">Total: ${this._people.length} personas</p>`
)}

when es lazy: la función que no se necesita no se evalúa.
Diferencia clave respecto a un ternario normal.

Si tus templates son simples, un ternario normal es perfectamente válido

guard()

Memorización de fragmentos costosos


import { guard } from 'lit/directives/guard.js';

${guard(
  [this._people, this._filterActive],
  () => {
    const list = this._filterActive
      ? this._people.filter(p => p.active)
      : this._people;

    return html`
      <ul>
        ${repeat(list, p => p.id, p => html`<li>${p.name}</li>`)}
      </ul>
    `;
  }
)}

Solo vuelve a ejecutar la función si alguna dependencia cambió.
Si otros estados del componente cambian sin afectar la lista,
el fragmento no se recalcula.

ifDefined()

Atributos opcionales — evita "undefined" como string


import { ifDefined } from 'lit/directives/if-defined.js';

html`
  <li>
    <a href=${ifDefined(person.profileUrl)}>
      ${person.name}
    </a>
    — ${person.age} años
  </li>
`

Sin ifDefined: si profileUrl es undefined,
el atributo href tendría el valor literal "undefined".

Con undefined: el atributo no se añade al elemento.
Con null: el atributo sí se añade. Solo elimina undefined.

Resumen

  ──────────────────────────────────────────────────────────────────
  repeat       lit/directives/repeat.js           Listas con keys
  classMap     lit/directives/class-map.js        Clases condicionales
  styleMap     lit/directives/style-map.js        Estilos inline dinámicos
  when         lit/directives/when.js             Condicional lazy
  guard        lit/directives/guard.js            Memorización
  ifDefined    lit/directives/if-defined.js       Atributos opcionales

No las uses por defecto — úsalas cuando tengas el problema que resuelven

Default slot

El mecanismo de composición nativo


import { LitElement, html, css } from 'lit';
import { customElement, property, state } from 'lit/decorators.js';

@customElement('my-accordion-item')
export class MyAccordionItem extends LitElement {
  @property({ type: String })
  heading = '';

  @state()
  private _open = false;

  render() {
    return html`
      <div class="accordion">
        <button class="trigger" @click=${this._toggle}>
          ${this.heading}
          <span class="icon">${this._open ? '▲' : '▼'}</span>
        </button>
        <div class="body" ?hidden=${!this._open}>
          <slot></slot>
        </div>
      </div>
    `;
  }

  private _toggle() {
    this._open = !this._open;
  }
}

Default slot

¿Cómo funciona la proyección?

Todo lo que el usuario coloca entre las etiquetas del componente
se proyecta en el punto donde está el <slot>.

El componente no necesita saber qué se le pasa:
párrafos, listas, imágenes, otros componentes — todo vale.

Los slots son nativos de la plataforma web, no una invención de Lit

Fallback content

Contenido por defecto cuando el slot está vacío


<div class="body" ?hidden=${!this._open}>
  <slot>
    <p class="empty-message">
      Este acordeón no tiene contenido.
    </p>
  </slot>
</div>

El contenido dentro de <slot> es el fallback.
Se muestra solo si el usuario no proyecta nada.

En cuanto el usuario añade contenido, el fallback desaparece automáticamente

Named slots

Proyectar contenido en posiciones específicas


render() {
  return html`
    <div class="layout">
      <header class="layout-header">
        <slot name="header"><h1>Título por defecto</h1></slot>
      </header>
      <aside class="layout-sidebar">
        <slot name="sidebar"></slot>
      </aside>
      <main class="layout-main">
        <slot></slot>
      </main>
      <footer class="layout-footer">
        <slot name="footer"><p>Pie de página por defecto</p></slot>
      </footer>
    </div>
  `;
}

Named slots — uso desde fuera

El atributo slot dirige el contenido


<my-slot-demo>
  <nav slot="header">
    <a href="/">Inicio</a>
    <a href="/about">Acerca de</a>
  </nav>
  <ul slot="sidebar">
    <li>Categoría 1</li>
    <li>Categoría 2</li>
  </ul>
  <article>
    <h2>Contenido principal</h2>
    <p>Este párrafo va al slot por defecto.</p>
  </article>
  <p slot="footer">Derechos reservados 2026</p>
</my-slot-demo>

El orden en el HTML del usuario no importa — cada elemento va al slot que le corresponde

Evento slotchange

Reaccionar cuando cambia el contenido proyectado


// En el template:
<slot @slotchange=${this._onSlotChange}></slot>

// Método:
private _onSlotChange(e: Event) {
  const slot = e.target as HTMLSlotElement;
  const nodes = slot.assignedNodes({ flatten: true });
  console.log('[slotchange] Nodos proyectados:', nodes.length);
}

slotchange se dispara cuando se añaden o eliminan elementos del slot.

slot.assignedNodes({ flatten: true })
Devuelve los nodos proyectados. flatten aplana slots anidados.

Útil para contar ítems, validar contenido obligatorio o actualizar aria-attributes

flatten: true — slots anidados

Cuando un componente con slot se proyecta dentro de otro


<!-- ComponenteA escucha slotchange en su slot -->
<componente-a>
  <mi-tarjeta>          <!-- también tiene su propio slot -->
    <p>Texto final</p>
  </mi-tarjeta>
</componente-a>

<!-- En el handler de ComponenteA: -->

// Sin flatten → [<mi-tarjeta>]
slot.assignedNodes()

// Con flatten → [<p>Texto final</p>]
slot.assignedNodes({ flatten: true })

Sin flatten: obtienes el componente intermedio (<mi-tarjeta>).
Con flatten: true: obtienes el contenido real, resolviendo todos los niveles.

En la práctica es poco frecuente — útil al componer componentes que se anidan varios niveles

::slotted()

Estilizar el contenido proyectado


.accordion { border: 1px solid #444; border-radius: 6px; overflow: hidden; }

.trigger {
  width: 100%;
  background: #2a2a2a;
  color: #fff;
  border: none;
  padding: 0.75rem 1rem;
  text-align: left;
  cursor: pointer;
  display: flex;
  justify-content: space-between;
}

.trigger:hover { background: #333; }
.body { padding: 1rem; background: #1a1a1a; }
.empty-message { color: #888; font-style: italic; }

::slotted(p)  { margin: 0 0 0.75rem; line-height: 1.6; color: #ddd; }
::slotted(ul) { padding-left: 1.25rem; color: #ddd; }

Solo afecta a los hijos directos del slot.
::slotted(p) no llega a párrafos dentro de un <div> proyectado.

Patrón de composición

Componentes contenedor (wrapper components)


<!-- El componente define la ESTRUCTURA -->
<my-accordion-item heading="¿Qué es Lit?">
  <p>Lit es una librería para crear Web Components...</p>
  <ul>
    <li>Ligero: 5 KB comprimido</li>
    <li>Estándar: Web Components nativos</li>
  </ul>
</my-accordion-item>

<!-- El acordeón sin contenido muestra el fallback -->
<my-accordion-item heading="Sección vacía"></my-accordion-item>

Patrón de composición

Principio fundamental

El componente define la estructura y el comportamiento.
El usuario decide el contenido.

Componentes que siguen este patrón:
acordeones, tarjetas, modales, layouts, tabs, tooltips...

Máxima reutilización — mínimo acoplamiento

Resumen

  <slot></slot>              Slot por defecto
  <slot>fallback</slot>      Se muestra si no hay contenido proyectado
  <slot name="x">            Slot con nombre
  slot="x"                   Dirige el contenido al slot correspondiente
  @slotchange                 Notifica cuando cambia el contenido
  slot.assignedNodes()        Nodos proyectados programáticamente
  ::slotted(selector)         Hijos directos proyectados (solo primer nivel)

Los slots son nativos del navegador — Lit simplemente los usa como corresponde

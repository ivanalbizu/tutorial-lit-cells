Reactive Controllers

Paso 10 — Lógica reutilizable en Lit

Encapsula comportamiento fuera del componente

¿Qué es un Reactive Controller?

Una clase que participa en el ciclo de vida del host


import { ReactiveController, ReactiveControllerHost } from 'lit';

class MiController implements ReactiveController {
  private host: ReactiveControllerHost;

  constructor(host: ReactiveControllerHost) {
    this.host = host;
    host.addController(this); // registro obligatorio
  }

  hostConnected()    { /* host se conectó al DOM   */ }
  hostDisconnected() { /* host se desconectó del DOM */ }
  hostUpdate()       { /* antes de cada actualización */ }
  hostUpdated()      { /* después de cada actualización */ }
}

El componente que lo usa se llama host

Interfaz ReactiveController

Cuatro hooks — todos opcionales

hostConnected — el host se conectó al DOM. Inicia recursos: timers, listeners.

hostDisconnected — el host se desconectó. Limpia recursos obligatoriamente.

hostUpdate — antes de cada render del host.

hostUpdated — después de cada render del host.

host.addController(this) en el constructor es obligatorio para activar los hooks.

ClockController

setInterval + limpieza automática


import { ReactiveControllerHost } from 'lit';

export class ClockController {
  private host: ReactiveControllerHost;
  private _intervalId: number | undefined;
  value = new Date();

  constructor(host: ReactiveControllerHost) {
    this.host = host;
    host.addController(this);
  }

  hostConnected() {
    this._intervalId = window.setInterval(() => {
      this.value = new Date();
      this.host.requestUpdate();
    }, 1000);
  }

  hostDisconnected() {
    clearInterval(this._intervalId);
    this._intervalId = undefined;
  }
}

host.requestUpdate() — el host no detecta cambios internos del controller automáticamente. Hay que notificarlo.

Sin clearInterval en hostDisconnected, el timer sigue corriendo aunque el componente ya no exista: fuga de memoria.

MouseController

Listener global con limpieza correcta


import { ReactiveControllerHost } from 'lit';

export class MouseController {
  private host: ReactiveControllerHost;
  position = { x: 0, y: 0 };

  // Arrow function guardada como propiedad para poder eliminarla
  private _onMouseMove = (e: MouseEvent) => {
    this.position = { x: e.clientX, y: e.clientY };
    this.host.requestUpdate();
  };

  constructor(host: ReactiveControllerHost) {
    this.host = host;
    host.addController(this);
  }

  hostConnected() {
    window.addEventListener('mousemove', this._onMouseMove);
  }

  hostDisconnected() {
    window.removeEventListener('mousemove', this._onMouseMove);
  }
}

La arrow function en propiedad garantiza la misma referencia en add y remove

FetchController<T>

Fetch con loading / error / abort


import { ReactiveControllerHost } from 'lit';

export class FetchController<T> {
  private host: ReactiveControllerHost;
  private _url: string;
  private _abortController: AbortController | undefined;

  loading = false;
  error: string | undefined;
  data: T | undefined;

  constructor(host: ReactiveControllerHost, url: string) {
    this.host = host;
    this._url = url;
    host.addController(this);
  }

  hostDisconnected() {
    this._abortController?.abort(); // cancela petición en vuelo
  }
}

FetchController — método fetch()


  async fetch() {
    this._abortController?.abort();
    this._abortController = new AbortController();

    this.loading = true;
    this.error = undefined;
    this.host.requestUpdate();

    try {
      const response = await fetch(this._url, {
        signal: this._abortController.signal,
      });
      if (!response.ok) throw new Error(`HTTP ${response.status}`);
      this.data = await response.json() as T;
    } catch (err) {
      if ((err as Error).name !== 'AbortError') {
        this.error = (err as Error).message;
      }
    } finally {
      this.loading = false;
      this.host.requestUpdate();
    }
  }

AbortError se ignora: la cancelación fue intencional, no un error real.

Uso en un componente

Controllers como class fields


import { LitElement, html } from 'lit';
import { customElement } from 'lit/decorators.js';
import { ClockController } from '../controllers/clock-controller.js';
import { MouseController } from '../controllers/mouse-controller.js';
import { FetchController } from '../controllers/fetch-controller.js';

interface Todo { id: number; title: string; completed: boolean; }

@customElement('my-controller-demo')
export class MyControllerDemo extends LitElement {

  // Tres controllers — Lit los registra al instanciarlos
  clock   = new ClockController(this);
  mouse   = new MouseController(this);
  fetcher = new FetchController<Todo>(
    this,
    'https://jsonplaceholder.typicode.com/todos/1'
  );

  render() { /* ... */ } // ver siguiente slide
}

Cada constructor llama host.addController(this) — el registro es automático

Class fields vs constructor

Dos formas de instanciar — una es idiomática en Lit


// ✗ En el constructor — verboso, innecesario
export class MyDemo extends LitElement {
  clock: ClockController;
  mouse: MouseController;

  constructor() {
    super();
    this.clock = new ClockController(this);
    this.mouse = new MouseController(this);
  }
}

// ✓ Como class fields — forma idiomática en Lit
export class MyDemo extends LitElement {
  clock = new ClockController(this);
  mouse = new MouseController(this);
}

Cada controller llama host.addController(this) en su constructor.
El registro ocurre automáticamente al instanciar el class field — sin código extra.

Se consumen como propiedades normales: this.clock.value, this.mouse.position, this.fetcher.data. Sin código extra para la reactividad.

Resumen

ReactiveController
  constructor(host)     → host.addController(this)
  hostConnected()       → inicia recursos
  hostDisconnected()    → limpia recursos (obligatorio)
  host.requestUpdate()  → notifica cambios al host

Uso en componente:
  clock   = new ClockController(this)
  mouse   = new MouseController(this)
  fetcher = new FetchController<T>(this, url)

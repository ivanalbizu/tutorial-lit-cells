Open Cells

Paso 12 — Scaffold de una app desde cero

Routing · Orquestación · Estado · Lazy loading

¿Qué hace Open Cells?

Cuatro responsabilidades principales

Routing — usa hash en la URL (#/home, #/about) para determinar qué página mostrar.

Orquestación de páginas — gestiona qué página está activa, cuáles están en caché y cuáles inactivas.

Estado compartido — canales de comunicación entre páginas sin dependencias directas.

Lazy loading — las páginas se descargan bajo demanda, solo cuando se visitan por primera vez.

Cada página es un Web Component independiente. Open Cells no sabe nada de ningún framework.

El atributo state

Tres valores posibles

state="active" — página visible actualmente.

state="cached" — página visitada, en DOM pero oculta.

state="inactive" — página que superó el límite de caché, se elimina del DOM.

Open Cells NO incluye el CSS para ocultar páginas. Si lo omites, todas las páginas se apilan visibles al mismo tiempo.

Definición de rutas

src/router/routes.ts


import { AppRoute } from '@open-cells/core/types';

export const routes: AppRoute[] = [
  {
    path: '/',
    name: 'home',
    component: 'home-page',
    action: () => import('../pages/home/home-page.js'),
  },
  {
    path: '/about',
    name: 'about',
    component: 'about-page',
    action: () => import('../pages/about/about-page.js'),
  },
];

Campos de cada ruta


  {
    path: '/',           // fragmento de URL (sin #)
    name: 'home',        // identificador para navegar
    component: 'home-page', // nombre del custom element
    action: () => import('../pages/home/home-page.js'),
    // ↑ import dinámico — habilita lazy loading
  },

action solo se ejecuta la primera vez que el usuario navega a esa ruta. El módulo se descarga bajo demanda.

El nombre del custom element en component debe coincidir con el argumento de @customElement en la clase.

Bootstrap con startApp

src/components/app-index.ts


import { startApp } from '@open-cells/core';
import { routes } from '../router/routes.js';

startApp({
  routes,
  mainNode: 'app-content',
});

Dos líneas bastan para arrancar Open Cells. Escucha los cambios de hash y carga el componente correspondiente.

Opciones adicionales: viewLimit controla cuántas páginas se mantienen en el DOM (por defecto 3). persistentPages excluye páginas del límite de caché.

Las páginas son componentes Lit

src/pages/home/home-page.ts


import { LitElement, html, css } from 'lit';
import { customElement } from 'lit/decorators.js';

@customElement('home-page') // coincide con routes.ts
export class HomePage extends LitElement {
  static styles = css`
    :host { display: block; padding: 2rem; }
    h1 { color: #1a1a2e; }
  `;

  render() {
    return html`
      <h1>Bienvenido a Cells App</h1>
      <p>Esta es la página de inicio.</p>
      <a href="#/about">Ir a About</a>
    `;
  }
}

Las páginas son Web Components normales. No hay ninguna clase base especial que deban extender.

Navegación con enlaces hash


<!-- Navegación declarativa: href con hash -->
<a href="#/about">Ir a About</a>
<a href="#/">Volver al inicio</a>

Open Cells intercepta los clics en estos enlaces automáticamente.

Para navegar programáticamente usa navigate('nombre-ruta') desde @open-cells/core, o this.pageController.navigate() dentro de una página con PageController.

viewLimit y persistentPages

Control del caché de páginas


startApp({
  routes,
  mainNode: 'app-content',
  viewLimit: 5, // máximo de páginas en DOM (por defecto: 3)
  persistentPages: [
    'home', // estas páginas nunca se eliminan del DOM
  ],
});

Las páginas que superan viewLimit reciben state="inactive" y se eliminan del DOM. Al revisitarlas se descargan e inicializan de nuevo.

persistentPages excluye páginas del límite. Se mantienen en DOM con state="cached" indefinidamente.

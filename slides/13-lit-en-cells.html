Componentes Lit en Cells

Paso 13 — Persistencia, reutilización y PageController

Tres roles que puede jugar un componente Lit

Tres roles en una app Cells

Cada componente tiene su lugar

Persistente — vive fuera del mainNode, en el HTML. Open Cells no lo toca. Ejemplo: <app-header>.

Reutilizable — Lit puro, sin dependencias de Open Cells. Se usa dentro de páginas como cualquier componente.

Página — vive dentro del mainNode. Open Cells gestiona su ciclo de vida. Puede usar PageController.

app-header — componente persistente

Fuera del router, usa navigate global


import { LitElement, html, css } from 'lit';
import { customElement } from 'lit/decorators.js';
import { navigate } from '@open-cells/core';

@customElement('app-header')
export class AppHeader extends LitElement {
  static styles = css`
    :host {
      display: block;
      background: #1a1a2e;
      color: white;
      padding: 0 1.5rem;
    }
    nav { display: flex; align-items: center; gap: 1rem; height: 56px; }
    button { background: none; border: none; color: white;
             font-size: 1rem; cursor: pointer; }
  `;

  render() {
    return html`
      <nav>
        <strong>Cells App</strong>
        <button @click=${() => navigate('home')}>Inicio</button>
        <button @click=${() => navigate('about')}>About</button>
      </nav>
    `;
  }
}

¿Por qué navigate importada y no PageController?

app-header no es una página. No vive dentro del mainNode. Open Cells no gestiona su ciclo de vida.

PageController necesita que el host sea una página gestionada por Cells. En app-header no tiene sentido.

Para componentes fuera del mainNode, importa navigate directamente desde @open-cells/core.

app-header en index.html

Fuera del contenedor mainNode


<body>
  <app-header></app-header>
  <div id="app-content"></div>
  <script type="module"
    src="/src/components/app-index.ts"></script>
</body>
        

// app-index.ts — importar para registrar el custom element
import { startApp } from '@open-cells/core';
import { routes } from '../router/routes.js';
import './app-header.js';

startApp({ routes, mainNode: 'app-content' });

Al colocarlo antes del div#app-content, el header no se ve afectado cuando el router cambia de página.

feature-card — componente reutilizable

Lit puro, agnóstico de Open Cells


import { LitElement, html, css } from 'lit';
import { customElement, property } from 'lit/decorators.js';

@customElement('feature-card')
export class FeatureCard extends LitElement {
  static styles = css`
    :host {
      display: block;
      border: 1px solid #e0e0e0;
      border-radius: 8px;
      padding: 1.25rem;
      background: white;
    }
    h3 { margin-top: 0; color: #1a1a2e; }
    p  { color: #555; line-height: 1.6; }
  `;

  @property() title = '';
  @property() description = '';
  @property() icon = '';

  render() {
    return html`
      ${this.icon ? html`<span>${this.icon}</span>` : ''}
      <h3>${this.title}</h3>
      <p>${this.description}</p>
    `;
  }
}

No importa nada de Open Cells. Podría usarse en cualquier proyecto web.

PageController

Reactive Controller para páginas Cells


import { LitElement, html, css } from 'lit';
import { customElement, state } from 'lit/decorators.js';
import { PageController } from '@open-cells/page-controller';

@customElement('demo-page')
export class DemoPage extends LitElement {

  // Se instancia igual que cualquier Reactive Controller
  private pageController = new PageController(this);

  @state() private _log: string[] = [];

  // ... ciclo de vida y render en slides siguientes
}

PageController se registra con addController internamente, igual que cualquier Reactive Controller.

onPageEnter y onPageLeave

Ciclo de vida de la página


  onPageEnter() {
    // Se llama cada vez que esta página se convierte en la activa
    this._log = [
      ...this._log,
      `[${new Date().toLocaleTimeString()}] Página entró`,
    ];
  }

  onPageLeave() {
    // Se llama cuando el usuario navega fuera de esta página
    this._log = [
      ...this._log,
      `[${new Date().toLocaleTimeString()}] Página salió`,
    ];
  }

La página no se destruye al navegar: queda con state="cached". Por eso onPageLeave es el lugar correcto para limpiar recursos.

Render de la página con feature-card


  render() {
    return html`
      <h1>Demo Page</h1>
      <div class="grid">
        <feature-card
          title="Routing"
          description="Open Cells gestiona la navegación con hash."
          icon="🗺"
        ></feature-card>
        <feature-card

          description="Las páginas se cargan bajo demanda."
          icon="⚡"
        ></feature-card>
        <feature-card
          title="PageController"
          description="Reactive Controller para la navegación."
          icon="🎮"

        Ir a About
      </button>
    `;
  }

navigate global vs pageController.navigate

Dos formas de navegar — misma función internamente


// Opción A: función importada — para componentes fuera del mainNode
import { navigate } from '@open-cells/core';
navigate('home');

// Opción B: método del controller — para páginas con PageController
this.pageController.navigate('home');

Ambas hacen lo mismo internamente. La diferencia es semántica: en páginas con PageController, usar el método del controller centraliza la dependencia de Open Cells.

El argumento es siempre el name de la ruta, no el path.

Registrar la ruta demo

src/router/routes.ts


export const routes: AppRoute[] = [
  { path: '/',      name: 'home',  component: 'home-page',
    action: () => import('../pages/home/home-page.js') },
  { path: '/about', name: 'about', component: 'about-page',
    action: () => import('../pages/about/about-page.js') },

  // Nueva ruta — cargada de forma lazy
  { path: '/demo',  name: 'demo',  component: 'demo-page',
    action: () => import('../pages/demo/demo-page.js') },
];

Al visitar #/demo por primera vez, el módulo se descarga. Al navegar y volver, el log acumula eventos porque la página no se destruyó.

Resumen

Persistente (fuera del mainNode)
  → usa navigate() de @open-cells/core

Reutilizable (Lit puro)
  → sin dependencias de Open Cells

Página (dentro del mainNode)
  → PageController = new PageController(this)
  → onPageEnter()  — página se activa
  → onPageLeave()  — página se oculta (state="cached")
  → this.pageController.navigate('nombre-ruta')

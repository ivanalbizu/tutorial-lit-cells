Open Cells construye la URL automáticamente
El hash resultante: #!/product/42

Leer parámetros en onPageEnter

Parsear window.location.hash


import { LitElement, html, css } from 'lit';
import { customElement, state } from 'lit/decorators.js';
import { PageController } from '@open-cells/page-controller';

@customElement('product-page')
export class ProductPage extends LitElement {
  pageController = new PageController(this);

  @state() productId: string = '';

  onPageEnter() {
    const hash = window.location.hash;
    const parts = hash.replace('#!', '').split('/');
    this.productId = parts[parts.length - 1] ?? '';
  }

  render() {
    return html`
      <h1>Producto ${this.productId}</h1>
    `;
  }

  static styles = css`:host { display: block; padding: 1rem; }`;
}

Leer parámetros en onPageEnter

El parse paso a paso


onPageEnter() {
  const hash = window.location.hash;   // "#!/product/42"
  const parts = hash.replace('#!', '').split('/');  // ["", "product", "42"]
  this.productId = parts[parts.length - 1] ?? '';   // "42"
}

Open Cells usa hash-routing
Toda la ruta vive en el fragmento de la URL
No hay servidor que la procese — todo ocurre en el cliente

El interceptor

Auth guard que se ejecuta ANTES de cada navegación

Usuario navega  →  interceptor()  →  ¿permitir?
                                         ↓ sí   →  página destino
                                         ↓ no   →  redirect a login

Recibe navigation (a dónde va) y ctx (contexto que tú controlas)
Devuelve si debe continuar o redirigir

Definir el interceptor

Rutas públicas y guardia de autenticación


const PUBLIC_ROUTES = ['home', 'login', 'product'];

startApp({
  pages: [ /* ... */ ],
  mainNode: 'app-content',
  interceptor: (navigation, ctx) => {
    const targetPage = navigation.to?.page || '';

    if (PUBLIC_ROUTES.includes(targetPage)) {
      return { intercept: false, redirect: '' };
    }

    if (!ctx.isAuthenticated) {
      return { intercept: true, redirect: { page: 'login' } };
    }

    return { intercept: false, redirect: '' };
  },
});

Definir el interceptor

Lógica del guardia

1. Ruta pública → dejar pasar
2. No autenticado → redirigir al login
3. Autenticado → dejar pasar

GOTCHA — redirect es un objeto

Error muy común en runtime


// INCORRECTO — redirect como string:
// return { intercept: true, redirect: 'login' };
// Open Cells hace redirect.page → undefined
// La navegación falla silenciosamente

// CORRECTO — redirect debe ser un objeto con page:
return { intercept: true, redirect: { page: 'login' } };

Si el redirect es un string, el router intenta hacer
redirect.page y obtiene undefined
La navegación falla sin ningún error visible

GOTCHA — skipNavigations no evita el interceptor

Dos funciones completamente distintas


startApp({
  pages: [ /* ... */ ],
  mainNode: 'app-content',
  // skipNavigations controla el HISTORIAL de navegación,
  // NO evita que el interceptor se ejecute.
  // Formato: cada entrada es { from?, to? }
  skipNavigations: [
    { from: 'login', to: 'home' },
  ],
  interceptor: (navigation, ctx) => { /* ... */ },
});

skipNavigations — esa transición no se añade al historial
(el botón atrás no regresará al login)

El interceptor se ejecuta igualmente.
Para rutas públicas, el interceptor debe retornar { intercept: false } explícitamente.

Patrón PUBLIC_ROUTES

Centralizar las rutas que no requieren autenticación


const PUBLIC_ROUTES = ['home', 'login', 'product'];

startApp({
  // ...
  interceptor: (navigation, ctx) => {
    const targetPage = navigation.to?.page || '';
    if (PUBLIC_ROUTES.includes(targetPage)) {
      return { intercept: false, redirect: '' };
    }
    // resto de la lógica de autenticación...
    if (!ctx.isAuthenticated) {
      return { intercept: true, redirect: { page: 'login' } };
    }
    return { intercept: false, redirect: '' };
  },
});

Un solo array controla qué páginas son públicas
Fácil de ampliar sin tocar la lógica del interceptor

updateInterceptorContext

Actualizar el contexto desde cualquier página


import {
  updateInterceptorContext,
  getInterceptorContext
} from '@open-cells/core';

// Al hacer login exitoso:
private handleLogin() {
  updateInterceptorContext({ isAuthenticated: true });
  this.pageController.navigate('home');
}

// Para comprobar el estado actual:
private checkAuth() {
  const ctx = getInterceptorContext();
  console.log('Autenticado:', ctx.isAuthenticated);
}

Login y página protegida

Implementación mínima


@customElement('login-page')
export class LoginPage extends LitElement {
  pageController = new PageController(this);

  private handleLogin() {
    updateInterceptorContext({ isAuthenticated: true });
    this.pageController.navigate('protected');
  }

  render() {
    return html`
      <h1>Login</h1>
      <button @click=${this.handleLogin}>Entrar</button>
    `;
  }
}

La página protegida no necesita lógica especial —
el interceptor ya la protege automáticamente

Resumen

Rutas dinámicas
  path: '/product/:id'        → dos puntos = parámetro
  navigate('product', { id }) → segundo argumento
  onPageEnter()               → parsear window.location.hash

Interceptor
  interceptor: (navigation, ctx) → se ejecuta ANTES de navegar
  PUBLIC_ROUTES                  → array de rutas sin guardia
  redirect: { page: 'login' }    → OBJETO, no string

Contexto
  updateInterceptorContext({ isAuthenticated: true })
  getInterceptorContext()

Los tres métodos de Channels

Cada función y su propósito


import { subscribe, publish, unsubscribe } from '@open-cells/core';

// PUBLICAR: enviar un nuevo valor al canal
publish('ch-cart', { items: [], total: 0 });

// SUSCRIBIRSE: recibir valores cuando el canal cambia
subscribe('ch-cart', (data) => {
  console.log('Carrito actualizado:', data);
});

// DESUSCRIBIRSE: dejar de escuchar el canal
unsubscribe('ch-cart');

El nombre del canal es una cadena arbitraria
Cualquier componente puede publicar o suscribirse

Patrón: onPageEnter / onPageLeave

Gestión correcta del ciclo de vida


@customElement('cart-page')
export class CartPage extends LitElement {
  pageController = new PageController(this);

  @state() cartItems: CartItem[] = [];

  onPageEnter() {
    subscribe('ch-cart', (items: CartItem[]) => {
      this.cartItems = items;
    });
  }

  onPageLeave() {
    unsubscribe('ch-cart');
  }
}

Patrón: onPageEnter / onPageLeave

Por qué es importante


export class CartPage extends LitElement {
  @state() cartItems: CartItem[] = [];

  onPageEnter() {
    // Nos suscribimos cuando la página se activa
    subscribe('ch-cart', (items: CartItem[]) => {
      this.cartItems = items;
    });
  }

  onPageLeave() {
    // Nos desuscribimos cuando la página se desactiva
    unsubscribe('ch-cart');
  }
}

Open Cells mantiene páginas en el DOM aunque estén ocultas
Sin unsubscribe en onPageLeave se acumulan
suscriptores duplicados → memory leaks

Flujo del carrito: publicar

product-page publica en el canal


import { subscribe, publish, unsubscribe } from '@open-cells/core';

@customElement('product-page')
export class ProductPage extends LitElement {
  pageController = new PageController(this);

  @state() private currentCart: CartItem[] = [];

  onPageEnter() {
    // También escuchamos para tener el estado actual
    subscribe('ch-cart', (items: CartItem[]) => {
      this.currentCart = items;
    });
  }

  onPageLeave() {
    unsubscribe('ch-cart');
  }

  private addToCart(product: CartItem) {
    const updated = [...this.currentCart, product];
    publish('ch-cart', updated);
  }
}

Leemos el estado actual → añadimos el nuevo producto → publicamos el array completo

Diagrama del flujo bidireccional

Las dos páginas leen y escriben en el mismo canal

product-page                    ch-cart                    cart-page
    │                              │                           │
    │  publish(updated)  ──────►   │   ──► callback(items) ►   │
    │                              │                           │  @state actualizado
    │                              │                           │  → re-render
    │  ◄── callback(items) ◄───    │   ◄──  publish(updated)   │
    │  (al eliminar un item)       │                           │

El canal es el punto de verdad compartido
Ambas páginas pueden leer y escribir en él
Se mantienen sincronizadas sin conocerse entre sí

Flujo del carrito: suscribirse

cart-page recibe y puede eliminar items


@customElement('cart-page')
export class CartPage extends LitElement {
  pageController = new PageController(this);
  @state() cartItems: CartItem[] = [];

  onPageEnter() {
    subscribe('ch-cart', (items: CartItem[]) => {
      this.cartItems = items ?? [];
    });
  }

  onPageLeave() {
    unsubscribe('ch-cart');
  }

  private removeItem(id: string) {
    const updated = this.cartItems.filter(item => item.id !== id);
    publish('ch-cart', updated);
  }

  private get total() {
    return this.cartItems.reduce((sum, item) => sum + item.price, 0);
  }
}

Ventajas de los Channels

Por qué usar Channels en lugar de otras soluciones

  • Desacoplado — las páginas no se importan entre sí, no se conocen
  • Global — cualquier componente en cualquier nivel puede publicar o suscribirse
  • Simple — sin reducers, sin acciones, sin selectores. Solo publish y subscribe
  • Lifecycle-aware — el patrón onPageEnter / onPageLeave evita memory leaks automáticamente

El canal es una cadena de texto — el contrato entre publicador y suscriptor
Si el nombre no coincide, los mensajes no llegan

Resumen

Channels — API
  publish('ch-cart', data)           enviar valor al canal
  subscribe('ch-cart', callback)     recibir valores
  unsubscribe('ch-cart')             dejar de escuchar

Patrón del ciclo de vida
  onPageEnter() → subscribe(...)
  onPageLeave() → unsubscribe(...)

Flujo del carrito
  product-page publica → cart-page escucha
  cart-page publica    → product-page escucha (si activa)
  Canal = fuente de verdad compartida

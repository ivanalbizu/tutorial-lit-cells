Controller como puente

Reactive Controller + Channels = sincronización global

Lit (hostConnected)    →  subscribe al canal
Lit (hostDisconnected) →  unsubscribe
Acciones del controller →  publish
Callback del canal     →  requestUpdate()

Resultado: cualquier componente con el controller
           se sincroniza automáticamente

El controller es el pegamento entre Lit y Cells
Los componentes solo instancian el controller y consumen sus datos

CartController — estructura

Implementa ReactiveController de Lit


import { ReactiveController, ReactiveControllerHost } from 'lit';
import { subscribe, unsubscribe, publish } from '@open-cells/core';

const CART_CHANNEL = 'ch-cart';

export interface CartItem {
  id: string;
  name: string;
  price: number;
  quantity: number;
}

export class CartController implements ReactiveController {
  private host: ReactiveControllerHost;
  items: CartItem[] = [];

  constructor(host: ReactiveControllerHost) {
    this.host = host;
    host.addController(this);
  }
}

CartController — ciclo de vida

hostConnected suscribe, hostDisconnected desuscribe


hostConnected() {
  subscribe(CART_CHANNEL, (items: CartItem[]) => {
    this.items = items ?? [];
    this.host.requestUpdate();
  });
}

hostDisconnected() {
  unsubscribe(CART_CHANNEL);
}

requestUpdate() — desencadena el re-render del componente host
Cada componente que instancia el controller queda sincronizado

CartController — acciones y getters

Las acciones siempre publican en el canal


add(product: Omit<CartItem, 'quantity'>) {
  const existing = this.items.find(i => i.id === product.id);
  let updated: CartItem[];
  if (existing) {
    updated = this.items.map(i =>
      i.id === product.id ? { ...i, quantity: i.quantity + 1 } : i
    );
  } else {
    updated = [...this.items, { ...product, quantity: 1 }];
  }
  publish(CART_CHANNEL, updated);
}

remove(id: string) {
  publish(CART_CHANNEL, this.items.filter(i => i.id !== id));
}

clear() {
  publish(CART_CHANNEL, []);
}

get total() {
  return this.items.reduce((sum, i) => sum + i.price * i.quantity, 0);
}

get count() {
  return this.items.reduce((sum, i) => sum + i.quantity, 0);
}

get isEmpty() {
  return this.items.length === 0;
}

items nunca se modifica directamente —
la actualización llega siempre a través del canal

Badge dinámico en app-header

Instanciar el controller = sincronización automática


import { CartController } from '../../controllers/cart-controller.js';

@customElement('app-header')
export class AppHeader extends LitElement {
  pageController = new PageController(this);
  cart = new CartController(this);

  render() {
    return html`
      <header>
        <span @click=${() => this.pageController.navigate('home')}>Tienda</span>
        <nav>
          <a @click=${() => this.pageController.navigate('cart')}>
            Carrito
            ${this.cart.count > 0
              ? html`<span class="badge">${this.cart.count}</span>`
              : ''
            }
          </a>
        </nav>
      </header>
    `;
  }
}

El header no sabe nada del carrito —
solo consume cart.count del controller

page-layout — uso en una página

<page-layout>

Slot sin nombre → contenido principal (<main>)
slot="subtitle" → cabecera contextual
slot="footer" → pie de página

Directivas en home-page

when(), classMap(), repeat()


import { repeat } from 'lit/directives/repeat.js';
import { when } from 'lit/directives/when.js';
import { classMap } from 'lit/directives/class-map.js';

render() {
  return html`
    <page-layout>
      <span slot="subtitle">Catálogo de productos</span>

      ${when(
        PRODUCTS.length > 0,
        () => html`
          <ul class="product-list">
            ${repeat(PRODUCTS, p => p.id, p => {
              const inCart = this.cart.items.some(i => i.id === p.id);
              return html`
                <li class=${classMap({ 'product-card': true, 'in-cart': inCart })}>
                  ...
                </li>
              `;
            })}
          </ul>
        `,
        () => html`<p>No hay productos disponibles.</p>`
      )}
    </page-layout>
  `;
}

@query para scroll automático

Referencia tipada a un elemento del Shadow DOM


@customElement('cart-page')
export class CartPage extends LitElement {
  cart = new CartController(this);
  @query('.cart-summary') summaryEl!: HTMLElement;

  onPageEnter() {
    // Esperar a que el template se haya renderizado
    this.updateComplete.then(() => {
      this.summaryEl?.scrollIntoView({ behavior: 'smooth' });
    });
  }

  render() {
    return html`
      <page-layout>
        <span slot="subtitle">Tu carrito (${this.cart.count} items)</span>
        ...
        <div class="cart-summary">
          <strong>Total: ${this.cart.total}€</strong>
          <button @click=${() => this.cart.clear()}>Vaciar carrito</button>
        </div>
      </page-layout>
    `;
  }
}

updateComplete — promesa que resuelve después del render
El scroll suave lleva al resumen cuando navegas al carrito

Diagrama: Controller como puente

Cada componente con el controller queda sincronizado

  app-header          home-page           cart-page
  cart = new          cart = new          cart = new
  CartController(this) CartController(this) CartController(this)
       │                   │                    │
       └───────────────────┴────────────────────┘
                           │
                    CartController
                    hostConnected() → subscribe('ch-cart', ...)
                    hostDisconnected() → unsubscribe('ch-cart')
                    add/remove/clear() → publish('ch-cart', ...)
                           │
                       canal ch-cart

Tres instancias del controller, un único canal
Cuando cualquiera publica, los tres se actualizan

Resumen

CartController
  hostConnected()    → subscribe + requestUpdate()
  hostDisconnected() → unsubscribe
  add/remove/clear() → publish en el canal
  total/count/isEmpty → getters calculados

page-layout
  slot sin nombre → contenido principal
  slot="subtitle" → cabecera contextual
  slot="footer"   → pie de página

Directivas en páginas
  when()     → renderizado condicional
  classMap() → clases dinámicas
  repeat()   → listas con clave única

@query
  .summaryEl → referencia tipada al DOM
  updateComplete.then() → scroll tras render

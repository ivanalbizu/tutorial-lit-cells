SearchController — debounce sin canal

Reactive Controller puro — solo setTimeout


export class SearchController implements ReactiveController {
  private host: ReactiveControllerHost;
  private timer: ReturnType<typeof setTimeout> | null = null;
  query: string = '';
  constructor(host: ReactiveControllerHost) {
    this.host = host;
    host.addController(this);
  }
  hostConnected() {}
  hostDisconnected() {
    // Limpiar el timer pendiente al desconectar
    if (this.timer) {
      clearTimeout(this.timer);
      this.timer = null;
    }
  }
  search(value: string) {
    if (this.timer) clearTimeout(this.timer);
    this.timer = setTimeout(() => {
      this.query = value;
      this.host.requestUpdate();
      this.timer = null;
    }, 300);
  }
  get hasQuery() {
    return this.query.trim().length > 0;
  }
}

El usuario deja de escribir 300ms → query se actualiza → re-render
hostDisconnected limpia el timer para evitar llamadas huérfanas

ToastController — auto-remove tras 3 segundos

Método estático para publicar desde cualquier parte


export class ToastController implements ReactiveController {
  private host: ReactiveControllerHost;
  messages: ToastMessage[] = [];

  hostConnected() {
    subscribe('toast-messages', (messages: ToastMessage[]) => {
      this.messages = messages ?? [];
      this.host.requestUpdate();
    });
  }

  hostDisconnected() { unsubscribe('toast-messages'); }

  static show(text: string, type: ToastMessage['type'] = 'info') {
    const id = Date.now().toString();
    publish('toast-messages', [{ id, text, type }]);
  }

  dismiss(id: string) {
    publish('toast-messages', this.messages.filter(m => m.id !== id));
  }
}

hostConnected — se suscribe al canal. Cada mensaje recibido actualiza el array y dispara un re-render.

static show() — publica en el canal sin instancia. Cualquier componente puede lanzar una notificación con una línea.

dismiss() — publica el array filtrado. El componente que tiene el controller instanciado recibe y renderiza el cambio.

WishlistController — segundo canal independiente

Mismo patrón, canal wishlist-items


export class WishlistController implements ReactiveController {
  private host: ReactiveControllerHost;
  items: WishlistItem[] = [];

  hostConnected() {
    subscribe('wishlist-items', (items: WishlistItem[]) => {
      this.items = items ?? [];
      this.host.requestUpdate();
    });
  }

  hostDisconnected() { unsubscribe('wishlist-items'); }

  toggle(product: WishlistItem) {
    const exists = this.items.some(i => i.id === product.id);
    publish('wishlist-items', exists
      ? this.items.filter(i => i.id !== product.id)
      : [...this.items, product]
    );
  }

  has(id: string) {
    return this.items.some(i => i.id === id);
  }

  get count() { return this.items.length; }
}

hostConnected / hostDisconnected — mismo patrón que CartController, canal wishlist-items.

toggle() — un único método: añade si el producto no está en la lista, lo elimina si ya está.

has(id) — devuelve true/false para que la tarjeta muestre el estado del botón de favorito.

product-card — tres controllers, un componente

Carrito, favoritos y toasts integrados


@customElement('product-card')
export class ProductCard extends LitElement {
  @property({ type: Object }) product!: { id: string; name: string; price: number };

  pageController = new PageController(this);
  cart = new CartController(this);
  wishlist = new WishlistController(this);

  private handleAddToCart() {
    this.cart.add(this.product);
    ToastController.show(`${this.product.name} añadido al carrito`, 'success');
  }

  private handleToggleWishlist() {
    const isInWishlist = this.wishlist.has(this.product.id);
    this.wishlist.toggle(this.product);
    ToastController.show(
      isInWishlist
        ? `${this.product.name} eliminado de favoritos`
        : `${this.product.name} guardado en favoritos`,
      'info'
    );
  }
}

El componente no gestiona ningún estado local —
todo el estado vive en los canales a través de los controllers

onPageLeave para lógica de salida

Más que desuscribirse: lógica de negocio al salir


@customElement('cart-page')
export class CartPage extends LitElement {
  cart = new CartController(this);

  onPageLeave() {
    if (!this.cart.isEmpty) {
      ToastController.show(`Tienes ${this.cart.count} item(s) en el carrito`, 'info');
    }
  }

  render() {
    return html`
      <h1>Tu carrito</h1>
      ${when(
        !this.cart.isEmpty,
        () => html`...lista de items...`,
        () => html`<p>Tu carrito está vacío.</p>`
      )}
    `;
  }
}

El CartController gestiona la suscripción automáticamente
via hostConnected / hostDisconnected
onPageLeave queda libre para la lógica de negocio
